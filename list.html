사이버보안 학습의 핵심 관점

  • 사이버보안은 처음부터 바로 공부할 수 있는 학문이 아니다.
  • 컴퓨터가 어떻게 설계되고 구현되었는지 이해해야 보안을 쌓을 수 있다.
  • 보안, 해킹, 컴퓨터 구조는 서로 분리되지 않고 연결되어 있다.
  • 하나의 분야에 대한 지식은 다른 영역에서도 활용된다.

컴퓨터 구조 기반 보안 공부 순서

  1. 컴퓨터 구조 이해
  2. 자료구조와 알고리즘
  3. 프로그래밍 언어 (Python, C 중심)
  4. 운영 체제
  5. 사이버보안 및 취약점 분석

보안 공부 철학 설명

기술은 빠르게 변화한다
- 10년 전 공격 기법이 현재는 쓸모없는 경우가 많다.
- 중요한 것은 기술 자체보다 그 기술이 등장한 이유이다.
창과 방패의 반복적 발전
- 공격과 방어는 서로 영향을 주며 발전한다.
- 하나를 이해하면 다른 하나의 구조도 보인다.
기본기와 실습 중심 학습
- 직접 코드를 작성하고 구조를 구현해보는 것이 중요하다.
- 실제로 분석하고 실습해야 이해가 깊어진다.
복합적인 지식의 중요성
- 컴퓨터 구조, 운영체제, 프로그래밍 지식은 서로 연결된다.
- 이러한 복합적 이해가 창의적인 보안 사고로 이어진다.
전문 분야 선택
- 기본기를 쌓은 후 자신에게 맞는 보안 분야를 선택해야 한다.
- 깊이 있는 전문성이 중요하다.

실습 기반 보안 학습 플랫폼

보안 공부는 이론에 머무르지 않고 실제로 코드를 보고 실습해보는 과정이 중요하다. 이를 위해 구조와 원리를 중심으로 학습할 수 있는 실습 기반 플랫폼을 활용하는 것이 도움이 된다.
